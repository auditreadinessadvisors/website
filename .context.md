# AI Working Context

## Project Summary
- Professional audit readiness and accounting advisory website.
- Static HTML5/CSS3 site with multiple language mirrors (English/Spanish).
- Features a dynamic 10-step Audit Readiness Assessment for lead generation.
- Responsive design system using CSS variables and Vanilla JS components.
- High SEO focus with comprehensive JSON-LD (Quiz, Organization, Service, FAQ).
- Performance-optimized with critical CSS and asset caching strategies.
- Security-first approach using CSP and strict HTTP headers via `_headers`.
- Zero-dependency core (using Chart.js only for assessment visualization).
- Hosted on a static platform (implied Cloudflare/Netlify based on `_headers`).
- Conversion-optimized with multiple lead capture gates and Calendly integration points.

## Ground Rules
- **Pure Javascript**: Prefer Vanilla JS for and ES6 modules for new logic; avoid adding heavy frameworks.
- **Content Changes**: Prefer modifying the static HTML files directly for content; ensure sync between EN and ES versions.
- **Assessment Logic**: Shared engine lives in `js/assessment-core.js`; locale data in `js/assessment.js` (EN) and `js/assessment-es.js` (ES).
- **Global Bootstrapping**: Use `js/main.js` for global behaviors (nav, scroll, language toggle).
- **Security**: Do not add inline scripts or external scripts without updating the CSP in `_headers`.
- **Paths**: Maintain the parallel directory structure for `es/` mirror pages.

## Where to Change Things
- **Global Navigation/Footer**: Edit `index.html`, `about.html`, etc., and the corresponding `es/` mirror files.
- **Animations/Interactions**: Update `js/main.js` (specifically the IntersectionObserver logic).
- **Assessment Engine**: Modify shared logic in `js/assessment-core.js`; locale-specific text in `js/assessment.js` (EN) or `js/assessment-es.js` (ES).
- **Styling/Theming**: Edit `css/style.css` (use the defined CSS variables in `:root`).
- **SEO/Metadata**: Update the `<head>` section of specific HTML files or the JSON-LD blocks.
- **Cache/Security Config**: Modify the `_headers` file in the root.

## Style Conventions
- **Naming**: BEM-lite for CSS (e.g., `nav__links`, `btn--primary`).
- **Formatting**: Semantic HTML5 tags; readable JS with function headers; camelCase for variables/functions.
- **Patterns**: Progressive enhancement using `IntersectionObserver` for visuals; functional approach for assessment state.
- **Language**: Any new feature must be implemented in both English and Spanish (`es/` folder).

## PR Checklist Before Final Answer
- Identify all files to change (Check EN and ES versions).
- Explain why each change belongs there (logic vs. content vs. style).
- Keep changes minimal and maintain the zero-dependency philosophy.
- Ensure no CSP violations (check external script/style calls).
- Validate JSON-LD syntax if schema blocks were touched.
- Confirm ES module imports are correct and paths are relative.
- Suggest a local test plan: Verify on mobile/desktop, check console for CSP errors, and test the assessment scoring path.
